1️⃣  DOES TRACE NEED GROUND TRUTH TRAJECTORIES FOR GlobalTargetPosGuidance?
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   
   ANSWER: NO ❌ Not needed
   
   WHY: GlobalTargetPosGuidance works by:
   • Computing distance between generated trajectories and target waypoint
   • Selecting the sample closest to waypoint during inference
   • The model GENERATES the trajectory to reach the waypoint
   
   PROOF: See guidance_loss.py line 673+:
   ```python
   class GlobalTargetPosLoss(GuidanceLoss):
       def forward(self, x, data_batch, agt_mask=None):
           # x: samples from diffusion (no GT needed)
           # Compute progress toward target
           local_target_pos = transform_points_tensor(self.target_pos, agent_from_world)
           # Only needs current state + target position
           progress_loss = compute_progress_loss(pos_pred, local_target_pos, urgency)
           return progress_loss
   ```
   
   IMPLICATION: You can use this for ANY scene, not just those with GT trajectories!


2️⃣  CAN YOU SYNTHESIZE TRAJECTORIES USING SPARSE WAYPOINTS?
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   
   ANSWER: YES ✅ Absolutely
   
   HOW:
   • Provide sparse waypoint(s): e.g., [(10.5, 0.0)]
   • Set urgency factor: 0.0 (slow) to 1.0 (fast)
   • Model generates smooth trajectory that reaches waypoint
   
   EXAMPLE:
   ```python
   guidance_config = {
       'name': 'global_target_pos',
       'weight': 1.0,                    # REQUIRED! Guidance strength
       'params': {
           'target_pos': [(10.5, 0.0)],  # Single waypoint
           'urgency': [0.8],             # 80% of max progress
           'pref_speed': 1.42,           # m/s
           'dt': 0.1,                    # timestep
           'min_progress_dist': 0.5,     # min progress/step
       },
       'agents': [0]
   }
   # Wrap as [[{config}]] for scenes
   guidance_config_list = [[guidance_config]]
   ```
   
   RESULT: Model generates natural trajectory that gradually approaches waypoint
   
   FOR MULTIPLE WAYPOINTS:
   • Create separate guidance configs for each waypoint
   • Or use GlobalTargetPosAtTimeLoss for time-constrained waypoints
   • Chain them sequentially during inference


3️⃣  DOES INFERENCE FRAME COUNT DEPEND ON ORCA DATASET?
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   
   ANSWER: NO ❌ Depends on config, not dataset
   
   WHAT CONTROLS IT:
   • SceneEditingConfig.num_simulation_steps = 100 (example)
   • This is set ONCE for all scenes
   • Each scene generates 100 steps regardless of dataset
   
   DIFFERENT SCENES HAVE:
   • Different valid frame ranges ✓ (but same inference horizon)
   • Different agent counts ✓
   • Different dynamics ✓
   • But same num_simulation_steps ✓
   
   TO CHANGE INFERENCE LENGTH:
   ```python
   cfg.num_simulation_steps = 200  # Now generate 200 steps instead of 100
   ```
   
   IMPLICATIONS:
   • You can use same config for ORCA, nuScenes, or Arena world
   • Inference length is INDEPENDENT of which dataset you use
   • Only controlled by your configuration

================================================================================
VALID GUIDANCE CONFIGURATIONS
================================================================================

All valid guidance types are in GUIDANCE_FUNC_MAP (guidance_loss.py, line 865):

✓ 'global_target_pos'           → Reach target at any time in future
✓ 'global_target_pos_at_time'   → Reach target at specific timestep
✓ 'target_pos'                  → Local frame version of global_target_pos
✓ 'target_pos_at_time'          → Local frame with time constraint
✓ 'agent_collision'             → Avoid other agents
✓ 'map_collision'               → Stay on roads/valid areas
✓ 'social_group'                → Keep agents together
✓ 'target_speed'                → Maintain specific speed
✓ 'min_speed'                   → Minimum speed constraint

COMBINING MULTIPLE GUIDANCE:
```python
guidance_config = [
    {
        'name': 'global_target_pos',
        'weight': 1.0,  # REQUIRED!
        'params': {'target_pos': [(10.5, 0.0)], 'urgency': [0.8], ...},
        'agents': [0, 1]
    },
    {
        'name': 'agent_collision',
        'weight': 1.0,  # REQUIRED!
        'params': {'num_disks': 5, 'buffer_dist': 0.2},
        'agents': None  # All agents
    }
```

================================================================================
KEY INSIGHTS & GOTCHAS
================================================================================

✓ DO:
  • Use GlobalTargetPosGuidance for waypoint following (recommended)
  • Set urgency = 1.0 for reaching waypoint ASAP
  • Set urgency = 0.5-0.8 for natural, leisurely movement
  • Enable guidance with class_free_guide_w > 0 (usually 1.0)
  • Use guide_clean=True for better quality
  • Use num_action_samples >= 20 for effective guidance

✗ DON'T:
  • Expect GT trajectories to be needed (they're NOT)
  • Use heuristic guidance configs (they need GT from sim_scene)
  • Set urgency > 1.0 (gets clamped, behaves unexpectedly)
  • Use class_free_guide_w = 0.0 (disables guidance completely)
  • Forget to wrap guidance config in list: [guidance_config]
  • Use mismatched num_agents in config vs actual scene

⚠️  COMMON ISSUES:
  • "Config validation failed" → Check 'name' matches GUIDANCE_FUNC_MAP
  • "Model not following waypoint" → Increase urgency or class_free_guide_w
  • "Out of memory" → Reduce num_action_samples or num_simulation_steps
  • "Trajectories too short" → Increase num_simulation_steps
  • "Dimensions don't match" → Verify target_pos shape = (num_agents, 2)
